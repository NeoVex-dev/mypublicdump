###### **#Non-Traditional Game Documentation**



Hey, I was recently thinking about making my game documents like the GDD and TDD and things like that, since I currently didn't have the recourses/equipment to directly start game development in Unreal Engine. I started doing some research about how to write the GDD, and although it seemed helpful for the game developers that used it, I felt something was missing. So, I decided to make my own type of master game document. My first decision was to make my documents in [AFFiNE](https://affine.pro/), so i had to flexibility of showing concepts visually if I wanted to. My second step was instead of writing long documents for how the game should be designed and have separate documents for things like level design and atmosphere, I decided to write the documents based on how the player experienced them. For example the starting of such a document would look like the following:

* 0.1 Splash Screen
* 0.2 Video-Intro Hook (only happens the first time the game is launched)
* 0.3 Title Screen
* 1.0 Tutorial
* 1.1 Intro
* 1.2 First layer
* 1.3 ........



You get the idea. I decided to name this master document the GECPD or Game Experience Cinematics and Progression Document. Although the name isn't the best, this structure helped me understand my own game better as well as build everything in m game player/experience first. Of course I do have separate dedicated documents like a features list and a style guide, but my master document for this game is the GECPD document structure I created.



##### ***Decision:* AFFiNE \& GECPD over Traditional GDD**

##### 

##### ***Reason:*** **Better visual flexibility and Experience-centric focus.**
